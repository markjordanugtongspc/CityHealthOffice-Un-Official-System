Local Network Hosting

Developer Guide for PHP & Vite Asset Syncing

Complete these steps whenever you switch to a different WiFi network. Since your Local IP Address changes per network, you must synchronize your configuration to allow external device connections.

1 Identify the Local IP

Launch the Vite development server via your terminal to broadcast the network path:

npm run dev

Locate the "Network" address in the terminal output:

âžœ Network: http://192.168.X.X:5173/
Pro Tip: Always ignore the "Local" address (127.0.0.1). Use the Network IP that matches your router's subnet (typically 192.168.x.x).

2 Update Configuration File

Navigate to config/vite_helper.php and update the VITE_HOST constant with the new IP identified in Step 1.

define('VITE_HOST', 'http://192.168.X.X:5173');

3 Execute Parallel Servers

The environment requires two active processes running in separate terminal instances:

  • Terminal A (Vite): Manages Tailwind processing & HMR via npm run dev
  • Terminal B (PHP): Executes server-side logic via php -S 0.0.0.0:8000

4 Remote Device Access

Ensure your secondary device (Phone/Tablet) is connected to the same WiFi network, then navigate to:

http://192.168.X.X:8000
Firewall Warning: If the connection times out, verify that the Windows Defender Firewall is either disabled or configured to allow inbound traffic on ports 8000 and 5173.
